// 03 — Roadmap Pas à Pas

Comment démarrer — les 7 étapes dans l'ordre

Chaque étape a un résultat visible et concret. Ne passe pas à la suivante avant d'avoir le résultat attendu de l'étape courante.

ÉTAPE 01

Créer l'application métier à monitorer

⏱ 2-3 jours ✓ Tu sais déjà faire ça Spring Boot

Avant de monitorer, il faut quelque chose à monitorer. Crée une petite app Spring Boot avec une logique métier réelle — ça rend le monitoring significatif et ton projet plus solide en démo.

  • Créer une API REST CRUD (ex: gestion de produits, utilisateurs, commandes)
  • Ajouter une base de données PostgreSQL ou H2
  • Créer au moins 3-4 endpoints (GET, POST, PUT, DELETE)
  • Ajouter une couche Service avec logique métier simple
  • Vérifier que l'app tourne sur http://localhost:8080
💡
Idée concrète : Une API de gestion de commandes e-commerce — Produits, Commandes, Paiements. Simple, réaliste, et parfait pour des métriques métier (taux de commandes, valeur moyenne, erreurs de paiement).
ÉTAPE 02

Ajouter Spring Boot Actuator + Logging JSON

⏱ 1 jour ✓ Facile Actuator · Logback · SLF4J

Deux ajouts dans pom.xml et quelques lignes de config. À la fin de cette étape, ton app expose déjà ses métriques et logue en JSON structuré.

  • Ajouter spring-boot-starter-actuator dans pom.xml
  • Configurer application.yml pour exposer les endpoints actuator
  • Vérifier http://localhost:8080/actuator/health → status UP
  • Ajouter logstash-logback-encoder pour des logs JSON
  • Ajouter des log.info() / log.error() dans tes Services avec MDC (requestId)
  • Vérifier que les logs en console sont au format JSON
// pom.xml — les 3 dépendances à ajouter <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-actuator</artifactId> </dependency> <dependency> <groupId>io.micrometer</groupId> <artifactId>micrometer-registry-prometheus</artifactId> </dependency> <dependency> <groupId>net.logstash.logback</groupId> <artifactId>logstash-logback-encoder</artifactId> <version>7.4</version> </dependency>
ÉTAPE 03

Prometheus + Grafana avec Docker Compose

⏱ 2-3 jours → Niveau moyen Docker · Prometheus · Grafana

Tu lances Prometheus et Grafana via Docker Compose. Prometheus commence à scraper les métriques de ton app. Tu vois tes premières courbes dans Grafana — c'est le moment "wow" du projet.

  • Installer Docker Desktop sur ta machine
  • Écrire docker-compose.yml avec prometheus + grafana
  • Configurer prometheus.yml pour scraper ton app (localhost:8080)
  • Vérifier dans Prometheus UI (port 9090) que ta target est UP
  • Ouvrir Grafana (port 3000), ajouter Prometheus comme datasource
  • Importer le dashboard "Spring Boot Statistics" (ID: 11378 sur grafana.com)
  • Générer du trafic sur ton API et voir les graphiques s'animer
⚠️
Attention : Prometheus ne peut pas scraper "localhost:8080" depuis l'intérieur d'un container Docker. Tu dois utiliser host.docker.internal:8080 (Mac/Windows) ou l'IP de ta machine (Linux) dans prometheus.yml.
ÉTAPE 04

Métriques métier custom avec Micrometer

⏱ 1-2 jours ✓ Facile avec Spring Boot Micrometer · Java

Les métriques par défaut (JVM, HTTP) sont bien. Tes métriques métier (commandes créées, erreurs de paiement, produits vus) sont ce qui rend ton projet UNIQUE. C'est là où tu montres ta valeur.

  • Injecter MeterRegistry dans tes Services Spring
  • Créer un Counter pour chaque événement important (commande créée, login échoué)
  • Créer un Timer sur tes opérations lentes (traitement de commande, appel externe)
  • Créer un Gauge pour des valeurs courantes (nb de commandes en attente)
  • Créer un dashboard Grafana custom avec tes métriques métier
  • Configurer une alerte Grafana si taux d'erreurs > 5%
ÉTAPE 05

Stack ELK — Centraliser les logs

⏱ 3-5 jours → Plus complexe Elasticsearch · Logstash · Kibana

C'est l'étape la plus lourde en configuration, mais aussi la plus impressionnante visuellement. Avoir une interface Kibana où tu cherches dans tous tes logs en temps réel, c'est très concret à montrer.

  • Ajouter Elasticsearch + Logstash + Kibana dans docker-compose.yml (⚠ 8Go RAM min)
  • Configurer le Logback TCP Appender pour envoyer les logs JSON à Logstash
  • Écrire le pipeline Logstash (input → filter → output elasticsearch)
  • Ouvrir Kibana (port 5601), créer un Index Pattern
  • Faire une recherche KQL (ex: level:"ERROR" AND service:"order-service")
  • Créer un dashboard Kibana avec histogramme d'erreurs par heure
💡
Alternative légère : Si ta machine est limitée en RAM, remplace ELK par Loki + Grafana (même Grafana que Prometheus). Loki est conçu pour les logs, consume 10x moins de mémoire qu'Elasticsearch, et s'intègre parfaitement avec Grafana.
ÉTAPE 06

Distributed Tracing avec Zipkin

⏱ 1-2 jours ✓ Relativement simple Zipkin · OpenTelemetry · Micrometer Tracing

Ajoute Zipkin en Docker. Configure Spring Boot pour envoyer les traces. Tu pourras visualiser la durée exacte de chaque opération au sein d'une requête HTTP — très utile pour l'optimisation.

  • Ajouter Zipkin dans docker-compose.yml
  • Ajouter micrometer-tracing-bridge-otel + opentelemetry-exporter-zipkin en pom.xml
  • Configurer management.tracing.sampling.probability=1.0
  • Faire une requête sur ton API et trouver sa trace dans Zipkin UI (port 9411)
  • Voir les spans (DB query, service call, etc.) avec leur durée
ÉTAPE 07

Reporting — Rapports automatiques planifiés

⏱ 2-3 jours ✓ Spring Boot pur @Scheduled · JasperReports · JavaMailSender

La cerise sur le gâteau. Un service Spring Boot qui génère automatiquement un rapport PDF ou envoie un résumé par email. C'est ce qui transforme ton projet technique en outil utilisable par un non-développeur.

  • Créer un @Scheduled service qui tourne toutes les 24h (ou à l'heure)
  • Récupérer les métriques via l'API Prometheus HTTP (ou depuis ta DB)
  • Générer un rapport PDF avec iText ou JasperReports (nb erreurs, latence, etc.)
  • Envoyer par email via JavaMailSender (Gmail SMTP ou MailHog en local)
  • Alternative : Générer un rapport Excel avec Apache POI

// 06 — Conseil Final

Par où commencer demain matin

Commence petit. Finit solide.

Ne cherche pas à faire la stack complète dès le début. Jour 1 : installe Docker Desktop, regarde 2 tutoriels, lance un container Nginx. Semaine 1 : crée ton app Spring Boot avec Actuator, vois le endpoint /actuator/health fonctionner. Semaine 2 : lance Prometheus et Grafana, importe un dashboard — tu auras déjà quelque chose de très concret.

L'erreur classique est de tout vouloir faire en même temps. Une étape → un résultat visible → étape suivante. En 6-8 semaines, tu auras un projet qui te différencie de la majorité des candidats en stage ou premier emploi, même ceux qui ont 2-3 ans d'expérience.

🔗
Ressources pour démarrer : spring.io/guides (guides officiels Actuator), hub.docker.com (images officielles), grafana.com/grafana/dashboards (dashboards à importer gratuitement), docs.spring.io/spring-boot/docs (référence Micrometer). Tout est gratuit, tout est en ligne.
